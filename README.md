# pgfreeplay-hub

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于收集、整理和发布多个独立的 HTML 页面。这些页面可以是工具类页面、信息展示页面、小游戏页面或其他任何形式的静态 HTML 文件。仓库本身不针对特定网站或服务，仅作为 HTML 资源的存档与分发点。

## 目录说明

```
.
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录对应一个独立的 HTML 页面项目，建议以页面功能或名称命名。
- `assets/`：存放页面间共享的静态资源，避免重复。
- `index.html`：如启用 GitHub Pages，可作为仓库的入口页面。

## 页面归档说明

- 每个 HTML 页面应尽量自包含，减少外部依赖。
- 如需引用外部资源（如 CDN 库），请注明来源版本。
- 页面文件名建议使用英文小写，单词间以连字符分隔。
- 归档时请附带简短的说明文件（如 `README.md`）描述页面用途。

## 维护说明

- 欢迎提交 Pull Request 增加新的 HTML 页面，或对现有页面进行改进。
- 新增页面请放置在 `pages/` 下相应的目录中，并确保页面功能完整、无恶意代码。
- 如发现页面无法正常显示或存在错误，请提交 Issue 说明情况。
- 仓库维护者将定期审查归档内容，清理无效或过时的页面。

## 如何使用

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件即可查看效果。
3. 如需部署到 GitHub Pages，请启用仓库的 Pages 功能，并将源目录设置为根目录。
